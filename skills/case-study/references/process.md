# Process — full step-by-step

The case study runs in 3 phases: Story extraction → Structure development → Output generation. Read this when running the skill end-to-end.

---

## Phase 1: Story extraction

**Purpose:** Extract narrative elements from provided inputs.

**Steps:**

1. **Step 1.1: Identify the transformation**
   - Before state: What was broken/painful
   - After state: What's now possible
   - Turning point: When they realized they needed change
   - **Output:** Transformation arc

2. **Step 1.2: Extract key quotes**
   - Challenge quote: Pain in their words
   - Solution quote: How they describe the product
   - Result quote: Outcome in their words
   - **Output:** Quote library (3-5 quotes)

3. **Step 1.3: Quantify results**
   - Primary metric: Headline number (e.g., "3-5x more content")
   - Secondary metrics: Supporting proof (e.g., "40-60 hours saved")
   - Timeframe: When results achieved
   - **Output:** Metrics table

4. **Step 1.4: Identify the "moment"**
   - Memorable story beat (e.g., "found Taylor Swift wearing their product")
   - Emotional hook that makes story shareable
   - **Output:** Signature moment

**Phase 1 checkpoint:** transformation arc documented; ≥3 quotes extracted; ≥2 quantified metrics; signature moment identified (if available).

---

## Phase 2: Structure development

**Purpose:** Organize story into case study sections.

**Steps:**

1. **Step 2.1: Write hero statement**
   - One sentence capturing the transformation
   - Should be memorable and shareable
   - **Output:** Hero statement (under 25 words)

2. **Step 2.2: Draft section content**
   - Company overview (2-3 sentences)
   - The challenge (problem + pain)
   - The solution (product + implementation)
   - The results (metrics + quotes)
   - **Output:** Section drafts

3. **Step 2.3: Create metrics callouts**
   - 3 headline metrics with labels
   - Format: [Number] + [Unit] + [Context]
   - **Output:** Metrics display format

4. **Step 2.4: Select pull quotes**
   - 1 quote per major section
   - Include attribution (name, title, company)
   - **Output:** Formatted quotes

**Phase 2 checkpoint:** hero statement complete; all sections drafted; metrics formatted; quotes attributed.

---

## Phase 3: Output generation

**Purpose:** Generate final case study in requested format.

**Steps:**

1. **Step 3.1: Choose output format**
   - Web page (HTML/Markdown)
   - PDF (structured for design handoff)
   - Sales deck slide (single slide summary)
   - **Output:** Format selected

2. **Step 3.2: Generate final content**
   - Apply format template
   - Include all sections
   - Add visual placeholders
   - **Output:** Complete case study

3. **Step 3.3: Create derivative assets**
   - Social proof snippet (for landing pages)
   - LinkedIn post version
   - Email snippet version
   - **Output:** Derivative content

**Phase 3 checkpoint:** primary format complete; derivatives created (if requested); all placeholders marked.


**Self-evaluation gate** (before review): hero statement under 25 words; ≥3 quantified metrics; ≥3 attributed quotes; no invented story beats or metrics. Mark `[METRIC NEEDED]` or `[CONFIRM: detail]` for verification.

**Review gate:** Level 2 (Spot Check). Chain into `sales-enablement` (case study slides), `landing-page-copy` (social proof snippets), `storytelling` (customer video campaigns).
