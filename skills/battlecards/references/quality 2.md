# Battlecards — Quality

Pre-delivery checklist + self-evaluation protocol. The body of SKILL.md keeps anti-hallucination guardrails and gotchas; this file holds the full evaluation runbook.

---

## Quality checklist (pre-delivery)

### Content quality
- [ ] All template sections present and filled
- [ ] Confidence levels on all claims
- [ ] Sources cited with access dates
- [ ] No invented competitor data, quotes, or metrics
- [ ] "When we lose" section is honest (not empty or dismissive)

### Sales readiness
- [ ] Scannable in 30 seconds during a call
- [ ] Counter-positioning is scripted (word-for-word usable)
- [ ] Objection responses have proof points
- [ ] Landmine questions are specific (not generic)
- [ ] Red flags help qualify out fast

### Actionability
- [ ] Sales rep can use this immediately in a competitive deal
- [ ] Talk tracks sound natural (not robotic)
- [ ] Feature comparison is balanced (honest about both sides)

---

## Self-evaluation protocol

After generating output, automatically run:

### Completeness check
- [ ] All template sections filled?
- [ ] Confidence levels assigned to all claims?
- [ ] Source citations included?

### Evidence quality check
- [ ] All competitive claims verified with sources?
- [ ] No fabricated quotes or testimonials?
- [ ] Win/loss patterns have supporting evidence?

### Self-roast questions
1. "Would a sales rep actually pull this up during a call?"
2. "Are the objection responses scripted enough to use verbatim?"
3. "Would the competitor's sales team recognize these claims as accurate?"
4. "Is the 'when we lose' section honest, or am I being defensive?"
5. "Would this battlecard survive a deal review?"

Based on evaluation, suggest improvements:
> "This battlecard could be stronger if [specific improvement]. Want me to [action]?"

## Worked example — what good looks like

A high-quality battlecard for "Acme HRIS vs Workday" includes:

- **Quick facts table**: Workday founded 2005, ~$8B revenue, 70K customers; Acme founded 2018, $40M ARR, 850 customers — both with [VERIFIED] tags and source URLs accessed in last 30 days
- **When we win**: "Mid-market companies (200–2K employees) under-served by Workday's enterprise focus" with three customer quotes naming the time-to-value gap
- **When we lose**: "Companies already on Workday Financials wanting unified suite" — honest acknowledgment, not minimized
- **Counter-positioning**: 4 specific scripts ("When they say 'enterprise-grade'…we say 'mid-market-tuned, no consultants required'")
- **Landmines**: "How long was your last Workday upgrade?" "Have you costed the consultant time for your last config change?"
- **Sources table**: 8 rows, each with URL + access date + confidence

## Anti-examples — what to flag

| Smell | Fix |
|-------|-----|
| "When we lose" is empty or contains only "rare edge cases" | Force at least 2 honest losing scenarios; sales reps stop trusting battlecards that pretend the competitor has no advantages |
| Pricing tier listed without source URL | Replace with `[CONFIRM: pricing — competitor's pricing page is gated, request via demo]` |
| Generic objection handler ("we're more reliable") | Replace with specific differentiator + proof point ("we're SOC2 Type II since 2022; their public listing shows Type I as of last audit") |
| Customer quote with no attribution | Either cite the source (G2 review URL, case study URL) or remove the quote |
| Landmine questions <3 per competitor | Add until ≥3; landmines are the most-used section in live calls |
| No access date on competitor data | Add `accessed YYYY-MM-DD` for every claim — battlecards go stale fast |
