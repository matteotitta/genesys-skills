# Search patterns by dimension

Detailed search query patterns for each research dimension. Use these exact patterns, substituting [competitor] with the actual company name/domain.

---

## 1. Company research

### Primary searches
```
"[competitor]" crunchbase funding
"[competitor]" tracxn company profile
"[competitor]" funding round series valuation
"[competitor]" raised million 2024 OR 2025
site:linkedin.com/company "[competitor]"
"[competitor]" team size employees
"[competitor]" revenue ARR
```

### Revenue-specific searches
```
"[competitor]" ARR annual recurring revenue
"[competitor]" revenue sacra
"[competitor]" "$[X]M ARR" OR "$[X] million ARR"
site:sacra.com "[competitor]"
site:getlatka.com "[competitor]"
```

### Disambiguation patterns
When company name is ambiguous, add qualifiers:
```
"[competitor]" [category] funding  # e.g., "Bolt.new AI coding funding"
"[competitor]" [founder name] startup
site:[competitor-domain] about
```

## Search best practices

### Query optimization
1. Use quotes for exact matches: `"Bolt.new"` not `Bolt.new`
2. Add year for recent data: `"[competitor]" funding 2025`
3. Use OR for alternatives: `"[competitor]" customers OR clients`
4. Exclude irrelevant with minus: `"Bolt" -ride-sharing -fintech`

### Domain qualification
When competitor name is common, use domain:
```
site:bolt.new
"bolt.new" (not just "bolt")
"stackblitz bolt.new"
```

### Source prioritization
1. Company website (highest authority)
2. Official press releases
3. Crunchbase, Tracxn, PitchBook
4. Analyst reports (Sacra, Contrary)
5. Tech publications (TechCrunch, etc.)
6. G2, Capterra (for reviews/pricing)
7. LinkedIn (for team/hiring)
8. Reddit, Twitter (lowest authority, verify claims)

### Handling paywalled sources
If a source is paywalled:
1. Check if snippet contains needed data
2. Search for the same info on free sources
3. Note "Source paywalled, unable to verify" if citing

### Date freshness
For time-sensitive dimensions, add date constraints:
```
"[competitor]" launch 2025
"[competitor]" funding after:2024-01-01
"[competitor]" review 2025
```
