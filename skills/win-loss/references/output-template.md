## Usage notes

### Reasons matrix

- Limit to 1-3 reasons per dimension — prioritize by frequency and impact
- Always include evidence count (X of Y calls)
- If no data for a dimension, write "Not discussed in transcripts"

### Insights format

- Lead with the pattern, then evidence
- Always include verbatim quote with attribution
- Confidence based on frequency: High (3+), Medium (2), Low (1)
- Opportunity must be actionable and specific

### Cross-reference analysis

- Only include if context documents provided OR firmographics extractable from transcripts
- Note "Insufficient data" if fewer than 3 calls per segment
- Highlight statistically meaningful patterns

### Verbatim library

- Organize by dimension AND outcome (win vs. loss)
- Include only quotes that directly support insights
- Provide brief context for each quote

### File naming

Save output as: `[client-name]-win-loss-analysis.md` or `[project-name]-sales-insights.md`
